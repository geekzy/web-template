Use the .sh scripts in the parent directory to perform various tasks:

- "./update.sh" is run on the web-template folder, and will update all submodules connected to the project.
- "./publish.sh" will publish those changes to the web-template folder for future use.
- "./fork.sh" allows you to make a clean client project based on the web-template folder.
- "./fork-update.sh" will pull in any new updates from web-template to the client's project folder
